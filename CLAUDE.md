# HPC-IDS — 混凝土配比计算工具

高性能混凝土智能设计系统（High-Performance Concrete Intelligent Design System）。网页工具，帮助工程师根据三种优化模型计算混凝土配比。

## 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 框架 | React + Vite | 前端 UI 框架，Vite 负责开发构建 |
| 组件库 | Material UI (MUI) | Google Material Design 规范，提供现成组件 |
| 图表 | ECharts | 绘制级配曲线对比图 |
| 计算逻辑 | 纯 JavaScript | 三个计算模型全部在浏览器前端运行，无需后端 |
| 语言 | JavaScript (JSX) | 不使用 TypeScript |

**当前阶段**：纯前端，无后端，无数据库。所有数据在页面内处理，刷新后重置。

## 混凝土基本组成

```
混凝土 = 胶凝材料 + 水 + 骨料（砂 + 石）+ 外加剂
```

| 成分 | 具体材料 | 作用 |
|------|----------|------|
| 骨料 | 石子 + 砂子 | 骨架：提供体积稳定性和承载力 |
| 胶凝材料 | 水泥 + 粉煤灰 + 矿粉 | 胶水：包裹并粘结骨料 |
| 水 | — | 引发水化反应，让胶水凝固 |
| 外加剂 | 增强剂 + 减水剂 | 调节性能的辅助剂 |

**强度三大决定因素**：
1. **水胶比 W/B**：水胶比越小 → 浆体越致密 → 强度越高（核心）
2. **浆骨比**：浆体要好填满骨架空隙并额外包裹骨料表面
3. **界面过渡区 ITZ**：石子与浆体的接触面，是混凝土最薄弱的位置

## 核心计算模型（按执行顺序）

### 第一步：Fuller 模型 — 骨架优化

**目标**：计算砂石（粗细骨料）的最优比例，使堆积密度最大、空隙率最低。

```
P(d) = 100 × √(d / D_max)
```

| 变量 | 说明 |
|------|------|
| `P(d)` | 粒径 d 的累计通过百分比（%） |
| `d` | 当前粒径（mm） |
| `D_max` | 最大粒径（mm） |

**工程含义**：曲线拟合各组分。输入各级碎石和砂子的筛分数据，通过调整"砂石"配比，使各粒径的分布曲线与理想抛物线（n=0.5）的残差平方和最小。

**输入**：各级骨料的筛分数据数组（粒径 + 通过率）、最大粒径 D_max
**输出**：最优砂率、骨架级配曲线

---

### 第二步：A&A 模型 — 浆体填充（Andreasen & Andersen）

**目标**：计算粉体（水泥、粉煤灰、石粉等微米级粉料）填充骨架空隙的需求量。

```
P(d) = (d^q - d_min^q) / (D_max^q - d_min^q)
```

| 变量 | 说明 |
|------|------|
| `P(d)` | 粒径 d 的累计通过率（0–1） |
| `d` | 当前粒径 |
| `d_min` | 最小粒径（通常为粉体粒径） |
| `D_max` | 最大粒径 |
| `q` | 分布模数，范围 **0.22–0.36**，默认 0.25 |

**q 值的影响**：
- q 趋近 0.22 → 粉体/浆体越多 → 流动性越好（适合自密实混凝土）
- q 趋近 0.36 → 骨料占比越高 → 密实度越高

**工程含义**：体积填充引擎。在固定骨架下，确定需要多少水泥、粉煤灰和石粉才能达到预设的"紧密程度"。

**输入**：Fuller 骨架结果、q 值（默认 0.25）、各粉体粒径参数
**输出**：各粉体用量（kg/m³）

---

### 第三步：MAA 模型 — 强度预测（Multi-component Activity Analysis）

**目标**：预测不同龄期的抗压强度，并根据增强剂品牌和养护温度修正。

```
Strength(t) = C_base + Σ(Kᵢ × SCMᵢ)
```

| 变量 | 说明 |
|------|------|
| `Strength(t)` | 龄期 t 时的预测抗压强度（MPa） |
| `C_base` | 水泥基础强度贡献 |
| `Kᵢ` | 第 i 种辅助胶凝材料的活性系数（由增强剂品牌和养护温度决定） |
| `SCMᵢ` | 第 i 种辅助胶凝材料用量（粉煤灰、矿粉等） |

**预测节点**：2 天、3 天、28 天强度

**工程含义**：非线性回归预测引擎。根据增强剂品牌和养护温度，修正基础（水泥/粉煤灰/矿粉）的活性贡献率。

**输入**：各粉体用量、增强剂品牌和养护温度、环境温度
**输出**：各龄期预测强度（MPa）

---

### 完整计算流程（伪代码）

```javascript
function calculateMix(targetStrength, targetSlump, temp, materialsList) {
  // 第一步：Fuller 模型计算骨架
  const skeleton = fullerOptimization(materialsList);

  // 第二步：A&A 模型计算粉体填充（q=0.25 为默认值）
  const pasteVolume = aaFillingEngine(skeleton, { q: 0.25 });

  // 第三步：MAA 模型强度校验与增强剂配比
  const finalMix = maaStrengthPredict(pasteVolume, temp, targetStrength);

  return finalMix;
}
```

## 辅助功能模块

### 含水率自动修正（Moisture Adapter）

骨料（尤其是砂）含水会影响实际水量，需要自动修正：

```
实际投料量 = 设计干料量 / (1 - 含水率)
实际加水量 = 设计水量 - 骨料自带水量
```

**UI 要求**：提供高频更新的含水率输入框，修改后实时刷新所有结果。

### 级配可视化

- 单张图显示：实测曲线 vs Fuller 理论曲线 vs A&A 理论曲线
- 支持手动调砂率滑块，曲线实时更新
- 图表库：ECharts

### 实验反馈修正（Feedback Loop）

- 用户可输入实测强度（2天 / 3天 / 28天）
- 系统根据实测值反向修正 MAA 引擎中的 K 系数
- 目的：提高后续预测准确性

## 安全约束（硬性限制，违反则阻止输出并告警）

| 约束项 | 条件 | 告警说明 |
|--------|------|----------|
| 水胶比 W/B | 不超过用户设定上限（默认 0.38） | 超出则禁止生成配比 |
| 水泥用量下限 | 不低于 80 kg/m³ | 低于则告警（专家模式可解释） |
| 粉体总量下限 | 不低于 450 kg/m³ | 低于则提示"存在离析防冻等风险" |

## 原材料数据对象结构

每种原材料需包含以下字段：

```javascript
{
  // 物理属性
  psd: [],              // 粒径分布数组 [{ d, P }]
  crush_value: 0,       // 压碎值
  moisture: 0,          // 含水率（%）
  fine_content: 0,      // 含泥量（%）

  // 化学属性
  xrf: {                // 氧化物含量
    CaO: 0,
    SiO2: 0,
    Al2O3: 0

  // 成本属性
  unit_price: 0,        // 单价（元/吨）

}
```

## 三个模型的关系总结

| 模型 | 解决的问题 | 输入 | 输出 |
|------|----------|------|------|
| Fuller | 骨架怎么堆最密？ | 筛分数据、最大粒径 | 最优砂率、骨架级配曲线 |
| A&A | 浆体填多少不离析？ | 骨架结果、q 值 | 各粉体用量 |
| MAA | 水泥这么少强度够吗？ | 粉体用量、温度、增强剂 | 各龄期预测强度 |

> 物理（A&A）决定混凝土的下限（不离析、好泵送），化学（MAA）决定混凝土的上限（高强度、高耐久）。
